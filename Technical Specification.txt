# 2. Callback Data

Формат:

```text
module:action:entity_id
```

Примеры:

```text
market:open
market:country:ru
market:country:kz

plate:view:123

sale:create:123
sale:buy:123

auction:view:123
auction:bid:123
auction:history:123

balance:topup
balance:history

profile:view

admin:user:123
admin:plate:123
admin:auction:123

nav:back
nav:home
```

Максимальная длина callback_data:

```text
64 байта
```

# 4. Навигационный стек

Для каждого пользователя хранить:

```text
screen_stack JSONB
```

Пример:

```json
[
  "HOME",
  "MARKET",
  "MARKET_RU",
  "PLATE_VIEW:123"
]
```

При нажатии:

```text
⬅ Назад
```

выполняется:

```text
stack.pop()
```

и открывается предыдущий экран.

# 9. Планировщики APScheduler

Каждую минуту:

```text
истекшие резервы
окончание аукционов
anti-sniping
```

Каждый час:

```text
неотправленные уведомления
```

Каждый день:

```text
поиск неактивных пользователей
создание резервной копии
```

# 15. Принцип интерфейса

Почти все действия выполняются через:

```text
editMessageText()

editMessageMedia()

editMessageReplyMarkup()
```

Бот должен визуально восприниматься как мобильное приложение внутри Telegram.

Новые сообщения создаются только для:

* уведомлений;
* инвойсов;
* временного ввода пользователя.
